ما هو ApexYard؟

ApexYard هو إطار عمل مصمم لإدارة محفظة من المشاريع البرمجيات. يعمل كـ "مصنع" للمشاريع - لا تضيف المشروع إلى ApexYard، بل يتم إنشاء المشروع داخل ApexYard.

هدفه الأساسي: ضمان أن كل مشروع يوصل إلى مستوى الإنتاج بجودة عالية تحت منهجية تطوير برمجيات صارمة (SDLC).

دورة التطوير SDLC

التخطيط → التصميم → البناء → المراجعة → ضمان الجودة → النشر → المراقبة

بوابات العمل (Workflow Gates)

البوابة قبل التحقق
1 التصميم → البناء PRD معتمد، تذكرات موجودة
2 البناء → المراجعة الاختبارات تمر، >80% تغطية
3 المراجعة → الدمج مراجعة معتمدة، CI أخضر
4 الدمج → الإنجاز ضمان الجودة يverify جميع معايير القبول

معايير الكود

  • اسم الفرع: {type}/{TICKET-ID}-{description}
  • عنوان PR: type(TICKET): description
  • رسالةcommit: نوع ثم موضوع ثم تفاصيل
  • لا push مباشر لـ main - كل شيء عبر PR
  • اختبارات مطلوبة - >80% تغطية
  • مراجعة مطلوبة قبل الدمج
  • موافقة CEO مطلوبة لكل دمج

الخطوات السريعة

  1. افتح OpenCode في مجلد apexyard: 
    opencode /home/zeyad/learn/apexyard
  2. استخدم المهارات: 
    /setup                    ← إعداد，首次تشغيل
/start-ticket 42         ← إعلان ticket نشط
/code-review          ← مراجعة الكود
/approve-merge 42     ← تسجيل موافقة CEO
  3. استخدم الوكلاء: 
    Use Rex to review PR #42
Use Shield to review PR #42 for security
Use Guardian to audit dependencies

قواعد مهمة

  • مهم تذكر واحد في كل مرة - لا تبدأ Ticket جديد قبل انتهاء الحالي
  • مهم مراجعتان - وكيل + إنسان قبل الدمج
  • مهم Ticket أولاً - استخدم /start-ticket قبل أي تعديل كود
  • الاختياري مراجعة CEO - موافقة مدير الشركة للدمج
