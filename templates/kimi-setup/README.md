# Kimi Code CLI Hook Wiring Templates

These templates wire ApexYard's mechanical enforcement hooks into the Kimi Code CLI harness.

## Files

| File | Purpose |
|------|---------|
| `apexyard-dispatch.sh` | Dispatcher script that walks up from `$PWD` to find and exec the requested hook from `.apexyard/hooks/` or `.kimi-code/hooks/`. |
| `config.toml.example` | Example `~/.kimi-code/config.toml` entries that register ApexYard hooks for SessionStart, UserPromptSubmit, PreToolUse, and PostToolUse events. |

## Setup

1. Copy the dispatcher into your Kimi Code CLI hooks directory:

   ```bash
   mkdir -p ~/.kimi-code/hooks
   cp templates/kimi-setup/apexyard-dispatch.sh ~/.kimi-code/hooks/apexyard-dispatch.sh
   chmod +x ~/.kimi-code/hooks/apexyard-dispatch.sh
   ```

2. Merge the example hook entries into your Kimi Code CLI config:

   ```bash
   # If you don't have an existing config:
   cp templates/kimi-setup/config.toml.example ~/.kimi-code/config.toml

   # If you already have a config, append the [[hooks]] blocks manually,
   # or use a TOML-aware merge tool.
   ```

3. Open a new Kimi Code CLI session inside an ApexYard ops fork. The SessionStart hooks should run silently; if you see "ApexYard: ..." banners, they're working.

## Updating hooks

After editing files under `.apexyard/`, regenerate the generated tool directories:

```bash
bin/apexyard-sync-tool-dirs
```

This keeps `.kimi-code/` and `.claude/` in sync with the canonical `.apexyard/` source.

## Custom hooks

You can add your own hooks by placing them in `.apexyard/hooks/` and registering them in `~/.kimi-code/config.toml` the same way. Remember that `.kimi-code/` is regenerated by `bin/apexyard-sync-tool-dirs`, so put custom hooks in `.apexyard/hooks/` (or a non-synced location) rather than `.kimi-code/hooks/`.
